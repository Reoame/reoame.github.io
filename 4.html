GitHub部署静态网页

2025年1月16日 · 技术指南 GitHub

GitHub Pages是一个免费的静态网站托管服务，可以直接从GitHub仓库托管的静态网页生成一个网站。下面介绍一下如何使用GitHub Pages部署静态网页。

准备工作

  • 注册一个GitHub账号。
  • 创建一个GitHub仓库，用于存放静态网页文件。

配置GitHub Pages

  1. 进入你的仓库，点击Settings。
  2. 找到GitHub Pages选项。
  3. 选择Source的Branch为main（或master），然后点击Save。

上传静态网页文件

将你的静态网页文件上传到仓库的main分支上。你可以通过以下方式上传：

  • 直接拖拽文件到仓库页面。
  • 使用Git命令行工具推送文件。
# 初始化本地仓库
git init
git add .
git commit -m "Initial commit"

# 关联远程仓库
git remote add origin https://github.com/用户名/仓库名.git

# 推送代码
git push -u origin main

访问你的静态网页

当你的仓库被GitHub Pages托管后，你可以通过以下地址访问你的静态网页：

https://你的用户名.github.io/仓库名/

例如：https://reoame.github.io/

注意事项

你的静态网页的主页文件名必须为index.html，否则GitHub Pages无法识别。

如果你有自己的域名，可以将域名与你的静态网页绑定。具体方法请参考GitHub Pages官方文档。

总结

通过GitHub Pages部署静态网页，可以快速、免费、简单地搭建一个个人博客或个人网站。如果你有兴趣，可以尝试一下，看看它是否满足你的需求。
