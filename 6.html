将项目部署到GitHub的完整指南

2025年1月15日 · 部署教程 GitHub

📦 准备工作

在开始部署前，请确保完成以下准备：

  • 注册GitHub账号
  • 安装Git客户端
  • 准备项目文件

🚀 创建仓库与推送代码

初始化本地仓库并推送到GitHub：

# 初始化本地仓库
git init
git add .
git commit -m "Initial commit"

# 关联远程仓库
git remote add origin https://github.com/用户名/仓库名.git

# 推送代码
git push -u origin main

🌍 使用GitHub Pages部署

对于静态网站，推荐使用GitHub Pages：

在仓库设置中找到"Pages"选项，选择部署分支（如gh-pages） 
# 创建部署分支
git checkout -b gh-pages
git push origin gh-pages

⚠️ 注意事项

  • 确保.gitignore文件配置正确
  • 敏感信息不要提交到公开仓库
  • 定期维护README文档
